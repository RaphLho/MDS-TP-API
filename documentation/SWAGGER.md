# Swagger / OpenAPI Documentation

Swagger (maintenant OpenAPI) est un outil puissant pour documenter les APIs REST. Il permet de décrire, produire, consommer et visualiser des services web RESTful de manière standardisée.

## Qu'est-ce que Swagger/OpenAPI ?

### Définition
- Un langage de description d'interface (IDL) pour décrire les APIs RESTful
- Un ensemble d'outils pour générer de la documentation, du code client et du code serveur
- Une spécification ouverte et standardisée (OpenAPI Specification)

### Avantages principaux
1. **Documentation interactive**
   - Interface utilisateur intuitive
   - Tests des endpoints en direct
   - Visualisation claire des modèles de données

2. **Génération automatique**
   - Documentation toujours à jour
   - Génération de code client dans différents langages
   - Génération de code serveur et de tests

3. **Standardisation**
   - Format JSON/YAML standard
   - Compatible avec de nombreux outils
   - Facilite la collaboration entre équipes

## Installation et Configuration

### 1. Installation des dépendances
Pour utiliser Swagger dans votre projet, plusieurs options s'offrent à vous :

- Via NPM :
  ```bash
  npm install swagger-ui-express swagger-jsdoc
  ```
- Via Yarn :
  ```bash
  yarn add swagger-ui-express swagger-jsdoc
  ```
